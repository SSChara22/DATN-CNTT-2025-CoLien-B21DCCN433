Vẽ lại sơ đồ Use Case cho tác nhân Admin hệ thống e-commerce với yêu cầu:

TÁC NHÂN: Admin

USE CASE CHÍNH:
1. Đăng nhập (Login)
2. Đăng xuất (Logout)
3. Quản lý người dùng (User Management)
   - Extended: Tìm kiếm, Thêm, Sửa, Xóa, Xem thông tin, Phân quyền
4. Quản lý tất cả danh mục (All Category Management)
   - Extended: Tìm kiếm, Thêm, Sửa, Xóa, Xem thông tin dữ liệu
5. Quản lý đơn hàng (Order Management)
   - Extended: Tìm kiếm, Xem thông tin, Cập nhật trạng thái
6. Quản lý nhập hàng (Import Order Management)
   - Extended: Tìm kiếm, Thêm, Sửa, Xem thông tin đơn nhập hàng
7. Quản lý tin nhắn (Message Management)
   - Extended: Tìm kiếm hội thoại, Trả lời tin nhắn
8. Thống kê (Statistics)
   - Extended: Xem các loại thống kê (tổng quan, theo tháng, theo ngày, doanh thu, lợi nhuận, tồn kho)

QUAN HỆ QUAN TRỌNG:

**<<include>> Relationship:**
TẤT CẢ các use case quản lý (3, 4, 5, 6, 7, 8) PHẢI include "Đăng nhập"

Lý do: Tất cả API endpoints của Admin đều có middleware verifyTokenAdmin - yêu cầu đăng nhập trước.

**<<extend>> Relationship:**
Các extended use cases extend từ main use cases tương ứng.

YÊU CẦU:
- Sử dụng PlantUML hoặc Mermaid
- Thể hiện RÕ quan hệ <<include>> từ "Đăng nhập" đến TẤT CẢ use case quản lý
- Mũi tên nét đứt với stereotype <<include>>
- Sơ đồ thể hiện: Không thể thực hiện chức năng quản lý nào mà không đăng nhập trước

OUTPUT: Code PlantUML/Mermaid hoàn chỉnh để render sơ đồ.
