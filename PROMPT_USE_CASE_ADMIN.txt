Vẽ lại sơ đồ Use Case cho tác nhân Admin của hệ thống thương mại điện tử với các yêu cầu sau:

TÁC NHÂN:
- Admin (vai trò R1 hoặc R4 trong hệ thống)

CÁC USE CASE CHÍNH:

1. **Đăng nhập (Login)**
   - Admin đăng nhập vào hệ thống bằng email và password
   - Sau khi đăng nhập thành công, hệ thống cấp JWT token
   - Use case này là ĐIỀU KIỆN TIÊN QUYẾT cho TẤT CẢ các use case khác (trừ Đăng xuất)

2. **Đăng xuất (Logout)**
   - Admin đăng xuất khỏi hệ thống
   - Xóa token và session

3. **Quản lý người dùng (User Management)**
   - Core function: "Tìm kiếm, thêm, xóa, sửa người dùng"
   - Extended use cases:
     * Tìm kiếm người dùng
     * Thêm người dùng
     * Sửa người dùng
     * Xóa người dùng
     * Xem thông tin người dùng
     * Phân quyền người dùng

4. **Quản lý tất cả danh mục (All Category Management)**
   - Core function: "Tìm kiếm, xem, thêm, xóa, sửa dữ liệu"
   - Bao gồm quản lý: Sản phẩm (Product), Banner, Blog, Loại vận chuyển (TypeShip), Voucher, Nhà cung cấp (Supplier), AllCode
   - Extended use cases:
     * Tìm kiếm dữ liệu
     * Thêm dữ liệu
     * Sửa dữ liệu
     * Xóa dữ liệu
     * Xem thông tin dữ liệu

5. **Quản lý đơn hàng (Order Management)**
   - Core function: "Tìm kiếm, xem, cập nhật trạng thái đơn hàng"
   - Extended use cases:
     * Tìm kiếm đơn hàng
     * Xem thông tin đơn hàng
     * Cập nhật trạng thái đơn hàng

6. **Quản lý nhập hàng (Import Order/Receipt Management)**
   - Core function: "Tìm kiếm, xem, thêm, sửa đơn nhập hàng"
   - Extended use cases:
     * Tìm kiếm đơn nhập hàng
     * Thêm đơn nhập hàng
     * Sửa đơn nhập hàng
     * Xem thông tin đơn nhập hàng

7. **Quản lý tin nhắn (Message Management)**
   - Core function: "Tìm kiếm, trả lời tin nhắn hội thoại"
   - Extended use cases:
     * Tìm kiếm hội thoại
     * Trả lời tin nhắn

8. **Thống kê (Statistics)**
   - Core function: "Xem dữ liệu thống kê"
   - Extended use cases:
     * Xem thống kê tổng quan (count card)
     * Xem thống kê trạng thái đơn hàng
     * Xem thống kê theo tháng
     * Xem thống kê theo ngày
     * Xem thống kê doanh thu
     * Xem thống kê lợi nhuận
     * Xem thống kê tồn kho sản phẩm

QUAN HỆ QUAN TRỌNG:

**<<include>> Relationship:**
- TẤT CẢ các use case sau đều PHẢI include "Đăng nhập":
  * Quản lý người dùng
  * Quản lý tất cả danh mục
  * Quản lý đơn hàng
  * Quản lý nhập hàng
  * Quản lý tin nhắn
  * Thống kê

**Lý do:** Trong code, tất cả các API endpoints của Admin đều có middleware `verifyTokenAdmin` - yêu cầu phải đăng nhập và có token hợp lệ trước khi thực hiện bất kỳ chức năng nào.

**<<extend>> Relationship:**
- Các extended use cases extend từ các main use cases tương ứng (như trong sơ đồ cũ)

**Quan hệ trực tiếp:**
- Admin → Đăng nhập (trực tiếp)
- Admin → Đăng xuất (trực tiếp)
- Admin → Các main use cases (nhưng các main use cases này include Đăng nhập)

YÊU CẦU VẼ:

1. Sử dụng PlantUML hoặc Mermaid syntax
2. Thể hiện RÕ RÀNG quan hệ <<include>> từ "Đăng nhập" đến TẤT CẢ các use case quản lý
3. Sử dụng mũi tên nét đứt (dashed arrow) với stereotype <<include>> để thể hiện
4. Các extended use cases vẫn giữ nguyên quan hệ <<extend>> với main use cases
5. Sơ đồ phải thể hiện được: Không thể thực hiện bất kỳ chức năng quản lý nào mà không đăng nhập trước
6. Có thể nhóm các use case vào các package/group để dễ nhìn hơn

VÍ DỤ CẤU TRÚC:
```
Admin --> Đăng nhập
Admin --> Đăng xuất
Đăng nhập <.. Quản lý người dùng : <<include>>
Đăng nhập <.. Quản lý tất cả danh mục : <<include>>
Đăng nhập <.. Quản lý đơn hàng : <<include>>
Đăng nhập <.. Quản lý nhập hàng : <<include>>
Đăng nhập <.. Quản lý tin nhắn : <<include>>
Đăng nhập <.. Thống kê : <<include>>
```

OUTPUT: Code PlantUML hoặc Mermaid hoàn chỉnh để vẽ sơ đồ Use Case với quan hệ đúng.
